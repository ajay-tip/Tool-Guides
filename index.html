TIP Project Management Tool User Guide

Welcome to the TIP Project Management Tool User Guide. This document will walk you through the core functionalities of the system, focusing currently on the Overview and Task Tracking pages.

Note: Look for the 🖱️ Interactive symbol throughout this guide. These sections are fully functional simulations where you can click, filter, and practice using the tool directly within this page.

System Hierarchy

The tool is built on a simple three-tier hierarchy:

  • Project: Attributes: Project Name, Client Name, PCODE, Project Lead, Project Manager, Start/End Dates, Overall Status.
  • Contract Task: Attributes: Contract Task Name, Contract Task Number, Contract Language Reference.
  • Action Item: The individual units of work required to complete a task. A task can have multiple action items.
    Attributes: Action Item Name, Assigned Resource, Description/Notes, Status, Due Date.

Feedback & Support

At the top of the interface, you will find a Feedback button. If you face any issues while using the tool, encounter bugs, or have feature requests, please use this button to report them. Your feedback is sent directly to the development team and will be addressed as soon as possible.

Forms & Database Architecture

We are using Fillout.com for all our forms. This allows users to view and edit the underlying Airtable database directly without needing a paid Airtable Creator or Editor subscription.

1. Overview Page

The Overview page is your central hub for managing and viewing high-level details of all ongoing and past projects.

Viewing Project Details

When you land on the Overview page, you will see a list of projects on the left-hand side.

  • To view a project: Click on any project name from the list.
  • What you see: The main panel will update to display key details for the selected project, including the Project Name, Client Name, PCODE, Project Lead, Project Manager, Status, Start/End Dates, Events, and Meeting Notes.

Editing Project Information

If you need to update high-level details, you can easily do so using the built-in form.

  1. Select the project you wish to edit from the left-hand list.
  2. Click the blue Edit Project button located above the project details.
  3. A Fillout form will appear, allowing you to update fields such as Project Lead, Project Manager, Start/End Dates, and the current Status (e.g., In Negotiation, In Progress).
  4. Make your changes and click Submit.

Adding Meeting Notes

Keeping track of discussions is crucial. You can add and/or attach meeting notes directly to a project.

  1. Select the relevant project from the left-hand list.
  2. Scroll down slightly and click the blue Add Meeting Notes button.
  3. A form will open where you can type your notes and optionally attach files (drag & drop or browse).
  4. Click Submit to save the notes to the project.

2. Task Tracking Page

The Task Tracking page, specifically the "Action Items" view, provides a detailed look at every individual task across your projects.

Viewing and Filtering Action Items

This page lists all action items, typically grouped by the project they belong to.

  • Grouped View: By default, you'll see tasks organized under their respective project headings.
  • Filtering: Use the dropdown menus at the top to filter the list. You can filter by:
    • Resources: See tasks assigned to specific team members.
    • Project: Isolate tasks for a single project.
    • Status: View only tasks that are "Not Started," "In Progress," or "Complete."

Editing an Action Item

To update the status or details of a specific action item:

  1. Locate the action item in the list.
  2. Click the blue Edit Action Item button in the corresponding row.
  3. A Fillout form will open, allowing you to modify the Action Item name, Description, Notes, Status, assigned Resources, and add a due date if necessary.
  4. Click Submit to save your updates.

3. Automated Emails

The TIP Project Management Tool automatically sends email notifications to keep the team aligned:

  • New Action Item: When a new action item is created, the assigned Resource is emailed.
  • Resource Update: When the resource on an action item is updated, the newly assigned Resource is notified.
  • Status: Production: When the status of an action item is changed to "Production", the Production team is notified.
  • Status: Revision: When the status of an action item is changed to "Revision", the assigned Resource is notified.
  • Task Complete: When a task is fully complete, the Project Manager is notified.
